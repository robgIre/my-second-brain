OD Setup Guide

One-time setup to get My Second Brain running. Same steps whether you're on Mac or Windows. Once done, you just reconnect and go.

Why do I need an OD or Dev Server?

My Second Brain is a wrapper around Claude Code CLI, which only runs on remote servers (ODs or Dev Servers) — not on your regular Windows or Mac laptop. If you tried running it locally and got errors about pip, Flask, or "module not found", this is why. This guide covers OD setup. If you have a reserved Dev Server, SSH in with port forwarding (ssh -L 5151:localhost:5151 your-devserver-name) and follow from Step 4 onwards. This setup is a one-time thing. After that, you just reconnect and start the server — takes 30 seconds.

1

Check the On Demand dashboard

Go to your browser's address bar (where you'd normally type google.com), type od, and hit Enter. This opens the On Demand dashboard.

Look at the top of the page under "Current Instances":

  • If it says "You do not currently have any instances" — that's fine, move to Step 2
  • If you see an existing instance listed, click "Save & Release" next to it first, then move to Step 2
↓
2

Open a terminal on your laptop

On a Mac: Press Cmd + Space to open Spotlight, type Terminal, and hit Enter.

On Windows: Press Win + R, type cmd, and hit Enter.

You should see a black or white window with a blinking cursor — that's your terminal.

↓
3

Connect to your OD server

In your terminal from Step 2, type these two commands (or click Copy and paste them in):

cd
dev connect

A list of options will appear. Use your arrow keys to find and select:
"WWW+FBSource+Configerator (Hardware: Default) (VPNLess)"
Then hit Enter.

It will ask you to verify your identity — tap your YubiKey (the small USB security key), or open the Duo app on your phone, get a code, and type it in. Either method works.

Wait a moment. You're connected when you see something like:

[yourusername@12345.od ~]

This means you're now on a remote server, not your laptop. Everything you type from here runs on the OD. All of the next steps happen in this same terminal window.

↓
4

Check Python and Claude Code are installed

In the same terminal window, run these checks:

python3 --version
claude --version

If Python 3 shows a version number (e.g. Python 3.10.12) — you're good.

If not, Python is usually pre-installed on ODs. Try python3.10 --version or python3.9 --version — the command name may differ. If none work, message Rob.

If Claude Code shows a version number — you're good.

If not, install it in the same terminal window:

curl -fsSL https://cli.anthropic.com/install.sh | sh

After installing, type source ~/.bashrc and hit Enter, then try claude --version again.

↓
5

Authenticate internal tools

Still in the same terminal window — this lets your brain access your calendar, tasks, and other internal tools. Run this command:

jf authenticate

It will show an error with a URL — that's expected. Open that URL in your browser (https://www.internalfb.com/intern/jf/authenticate/), follow the steps, and it will give you a command to paste back into your terminal. It looks like:

jf auth 100034... <token>

Paste that command and hit Enter. When you see "success — Jellyfish is now authenticated" you're done.

This token expires periodically. If your calendar or tasks stop working later, just repeat this step.

↓
6

Install plugins (calendar, tasks, etc.)

Still in the same terminal window — without plugins, your brain can't access your calendar, tasks, GChat, or Workplace. Paste these commands:

claude plugin install meta@Meta
claude plugin install llm-rules@Meta
claude plugin install meta_knowledge@Meta
claude plugin install meta_codesearch@Meta

You only need to do this once per OD. If you release your OD and get a new one, you'll need to install plugins again.

↓
7

Install and start My Second Brain

Still in the same terminal window — first, set up the proxy so your OD can reach GitHub:

export https_proxy=http://fwdproxy:8080
export http_proxy=http://fwdproxy:8080

Then, in the same terminal window, paste this to download and start the app. Takes about 30 seconds:

git clone https://github.com/robgIre/my-second-brain.git && cd my-second-brain && bash setup.sh

You'll see a green URL when it's ready. Don't close this terminal tab — it's running the server. If you close it, the app stops.

↓
8

Open the dashboard

You're done with the terminal! Open your browser and go to:

http://localhost:5151

Click the green "Connect Brain" button. When it says "Your brain is active" — you're all set.

Page won't load? ("This site can't be reached") This is the most common issue. It means port 5151 isn't being forwarded from the OD to your laptop. Here's how to fix it: 1. Don't close the terminal tab where the server is running (Step 7) 2. Open a new terminal tab on your laptop — press Cmd+T (Mac) or open a new Command Prompt (Windows) 3. In the new tab, paste this command (replacing the hostname with yours): ssh -L 5151:localhost:5151 yourusername@YOUR-OD-NUMBER.od.fbinfra.net 4. Open the Duo app on your phone, get a code, and type it in (or tap your YubiKey) 5. Try http://localhost:5151 in your browser again Your OD number is in your terminal prompt from Step 3 — e.g. yourusername@987654.od means your hostname is 987654.od.fbinfra.net.

↓
9

Set up your profile

Click "About Me" in the sidebar. Fill in your name, role, team, manager, and location. Write a short background about what you do. Click "Save & Sync".

This teaches your brain who you are so it has context every time you use it.

You're done! From now on, everything happens in your browser. You won't need to touch the terminal again unless you're restarting or updating (see below).

Troubleshooting

"This site can't be reached" / "localhost refused to connect"

This is the most common issue. It means port 5151 isn't being forwarded from the OD to your laptop.

Fix:
1. Don't close the terminal tab where the server is running
2. Open a new terminal tab on your laptop — press Cmd+T (Mac) or open a new Command Prompt (Windows)
3. In the new tab, paste this (replacing the hostname with yours):
ssh -L 5151:localhost:5151 yourusername@YOUR-OD-NUMBER.od.fbinfra.net
4. Open the Duo app on your phone, get a code, and type it in (or tap your YubiKey)
5. Once connected, try http://localhost:5151 in your browser again

Your OD number is in your terminal prompt — e.g. yourusername@987654.od means your hostname is 987654.od.fbinfra.net.

"Calendar not working" / "I don't have access to your calendar"

This means plugins aren't installed on your OD, or the server wasn't restarted after installing them.

Fix:
1. Open a new terminal tab on your laptop — press Cmd+T (Mac)
2. Connect to your OD from this new tab:
ssh yourusername@YOUR-OD-NUMBER.od.fbinfra.net
3. Paste the plugin install commands:
claude plugin install meta@Meta
claude plugin install llm-rules@Meta
claude plugin install meta_knowledge@Meta
claude plugin install meta_codesearch@Meta
4. Go back to the terminal tab running the server and press Ctrl+C to stop it
5. Type exit and hit Enter to disconnect from the OD
6. Reconnect to your OD the same way you did during setup (Step 3)
7. Start the server again: cd ~/my-second-brain && bash setup.sh

Do NOT run claude plugin install from inside the My Second Brain dashboard — you'll get a "nested session" error.

"Port already in use"

This means the app is already running in another terminal tab, or it didn't shut down properly last time.

Fix:
1. Check your other terminal tabs — if you see the server running somewhere, press Ctrl+C in that tab to stop it
2. If you can't find it, start on a different port instead. In the terminal tab where you're connected to your OD, run:
cd ~/my-second-brain && MSB_PORT=5152 bash setup.sh
3. Then open http://localhost:5152 in your browser (note: 5152, not 5151)

"Claude Code CLI not found"

Claude Code isn't installed on your OD.

Fix: In the terminal tab where you're connected to your OD, run:
curl -fsSL https://cli.anthropic.com/install.sh | sh
Then type source ~/.bashrc and hit Enter.
Verify it worked: claude --version

"Couldn't connect to server" when cloning from GitHub

ODs can't reach GitHub directly — you need to set the proxy first.

Fix: In the terminal tab where you're connected to your OD, run:
export https_proxy=http://fwdproxy:8080
export http_proxy=http://fwdproxy:8080
Then retry the git clone command from Step 7.

If that doesn't work, try this proxy instead:
export https_proxy=http://fwdproxy.any.facebook.com:8080

"Connect Brain" fails

Claude Code may not be responding on your OD.

Fix:
1. Open a new terminal tab — press Cmd+T (Mac)
2. Connect to your OD: ssh yourusername@YOUR-OD-NUMBER.od.fbinfra.net
3. Run claude --version to check if Claude Code is installed
4. If it's not installed, follow the "Claude Code CLI not found" fix above
